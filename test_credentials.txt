## How to Test Each Functionality

### Login Page
- URL: http://localhost:3000/signin
- Use any of the above credentials

### Dashboard Access
- After login, you'll be redirected to the dashboard
- Different roles will see different menu options

### Content Management
- Encoder: Can upload content
- Editor: Can review content
- Approver: Can approve content
- Admin: Can manage users and content

### Role Management
- Super Admin/Admin: Can access user and role management
- URL: http://localhost:3000/dashboard/users/roles

## Troubleshooting Tips

1. If you get an authentication error:
   - Make sure you're logged in with valid credentials
   - Check that your session hasn't expired

2. If a feature is not accessible:
   - Verify you're using an account with appropriate permissions
   - Some features are restricted to specific roles

3. To access the Django Admin Interface:
   - URL: http://127.0.0.1:8000/admin/
   - Use the superuser account (admin)
   
## Important Notes

- The roles and permissions are controlled by Django groups
- Each role has specific permissions as defined in the system
- Always log out when testing different roles to ensure clean sessions
- The system now has all required role groups: Encoder, Editor, Approver, Admin, and Super Admin
